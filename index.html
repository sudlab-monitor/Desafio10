Monitor Inteligente da Mobilidade

Essa interface é uma proposta de Solução Inovadora em processo (build-up) para o Desafio 10. Saiba mais nesse vídeo.

Ela renderiza datasets com geomapeamento do data.rio através de snippets em linguagem Python com a biblioteca geopandas; é totalmente funcional offline e em ambientes controlados. Está preparada para se tornar um API a servico da SMTR, do Escritório de Dados, e também para ser integrada com dados GTFS em tempo real; sua iteração pode gerar outros monitores e aplicativos vinculados a PCRJ. Os arquivos estão disponíveis em https://github.com/sudlab-monitor/Desafio10.

Nesse primeiro momento, decidimos demonstrar que é possível, em uma interface HTML com código .js, sem plug-ins, totalmente aberta e auditável, e que usa datasets Shapefile armazenados e compartilhados no datalake, visualizar todos os modais sob a alçada da SMTR e inclusive os que estão no território da Cidade do Rio de Janeiro mas que são regulados por outras esferas de governança, como o Metrô Rio.

A principal inovação que propomos é o formato de visualização de dados dos modais considerando ambientes controlados. Para compreender essa inovação, é importante abrir a interface sem conexão a Internet. O arquivo .png, aparentemente vinculado a um HTML, na verdade só é gerado no momento em que a página é carregada no browser: nao há hyperlinks no código dos mapas, na própria página do GitHub nao há arquivos de imagens, e ainda assim a visualização de dados é 100% funcional offline. Isso se dá porque os dados disponibilizados no datalake são processados previamente dentro do próprio código HTML, usando snippets .py dentro do ambiente Jupyter Notebook, e renderizados quando a página é carregada.

O SudLab recomenda um projeto de Engenharia de Dados que permita a integração de todas as data sources de interesse da PCRJ para viabilizar possibilidades de publicação (deployment) em outras estruturas de dados que não usem somente GCP, conforme solicitado no Desafio 10, para posterior desenvolvimento do sistema de alarmes a ser monitorado.

Todas estas questões evidenciam que o Desafio 10 proposto necessita não da compra de um produto, mas de uma Contratação de Prestação de Serviços para Solução Inovadora (por exemplo, CNAE 8599-6/03 - Treinamento em Informática), que possa estruturar uma equipe dedicada em tempo integral a essa entrega e que possa também desenvolver um eventual Monitor Inteligente que integre outras Soluçoes Inovadoras. A equipe de Análise de Dados do SudLab está totalmente disponível para avançar nesse diálogo e iremos acompanhar a possível publicação de um Edital de CPSI, no intuito de contribuir com critério científico ao ecossistema de inovação carioca.
